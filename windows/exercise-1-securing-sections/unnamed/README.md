Create a process that creates an anonymous section with read-write access while simultaneously passing a strict SECURITY_ATTRIBUTES. Create a second process and have the first process duplicate a read-only handle to the section into the second process. From the second process, attempt to obtain a read-write handle to the same section (for example by duplicating/reopening the object as permitted by its security descriptor) and verify whether writes are possible. Attach to both processes using Process Explorer to inspect the handles, observing how the strict descriptor you applied was entirely discarded by the kernel simply because the object lacked a name. Then apply a restrictive DACL to the section and repeat the test to observe how the DACL prevents acquisition of stronger access rights. This exercise is based on James Forshaw's discussion of anonymous sections and their default security characteristics.
